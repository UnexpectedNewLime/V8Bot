# Codex Project Context

Before making changes in this repository, read the local Codex rules at
`.codex/instructions.md` if that file exists. Also scan `.codex/commands/` and
`.codex/skills/` when the task mentions a local command, workflow, or skill.

The `.codex/` directory is intentionally gitignored because it may contain
machine-local preferences, scratch commands, and operational notes. Treat it as
readable project context, but do not commit its contents.
